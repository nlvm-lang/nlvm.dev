Get started

From a clean machine to a running NL program in about a minute. The toolchain is three binaries: nlc the compiler, nlvm the virtual machine, and nltest the conformance test runner.

1. Install nlvm

The fastest way, for most people — installs prebuilt nlc and nlvm binaries, no Rust toolchain required:

$ curl -fsSL https://nlvm.dev/install.sh | bash

Prebuilt binaries are published for Linux (x86_64) and macOS (Apple Silicon). It installs into ~/.local/bin — make sure that's on your $PATH.

Building from source

On another platform, or want to build it yourself? You need a Rust toolchain. Then:

$ git clone https://github.com/nlvm-lang/nlvm.git
$ cd nlvm
$ cargo build -r
$ ./install.sh

Run from inside the cloned repo, install.sh detects the source checkout and symlinks the binaries it just built into ~/.local/bin instead of downloading one.

2. Write a program

Create Main.nl:

namespace hello;

class Main {
    public static int main(string[] args) {
        system.Out.println("Hello, world!");
        return 0;
    }
}

3. Compile and run

nlc compiles your sources into a single .nlp program file:

$ nlc Main.nl -o Main.nlp
$ nlvm Main.nlp
Hello, world!

Point -o at a directory instead and the file is named after the class that defines main:

$ nlc Main.nl -o out/
$ nlvm out/hello.Main.nlp
Hello, world!
A .nlp file bundles every module of your program — it's the only artifact you ship. Prefer one .nlm module per class? nlc --emit-modules Main.nl -o out/ gives you the exploded layout, and nlvm out/ loads a whole directory.

4. Check your setup

$ nlc --version
{{NLC_VERSION_LINE}}
$ nlvm --version
{{NLVM_VERSION_LINE}}

Both report the crate version and the release of the language specification this build implements.

5. Run the test suite (optional)

The repository ships a YAML conformance suite — one file per language feature — which is what CI runs:

$ cargo test --workspace
$ cargo run -p nl-test-runner -- tests
{{TEST_SUMMARY}}

Each test is a plain YAML file: NL source files, expected stdout, expected exit code. They double as executable documentation — browse them to see every feature in action.
